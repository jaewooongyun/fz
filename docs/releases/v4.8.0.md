## Highlights

### 1. Cargo-Cult Import 3중 다층 가드 (Phase 1, 본 사례 차단)

**작성 시점** (impl-correctness): 새 파일 작성 시 각 import에 대해 *사용 심볼* grep 검증. 0건이면 마찰 신호 보고.

**리뷰 시점** (review-quality + fz-review SKILL.md 검증 4-E 항목 7): 양방향 Symbol Coverage — 추가 방향 검증 신설.

**컴파일 시점** (TVING `.swiftlint.yml`): `unused_import` rule schema-level 차단 (사용자 `brew install swiftlint` 후).

`fz-code/SKILL.md` 마찰 신호 카탈로그 31번째 항목으로 "Redundant Import" 추가:

| Redundant Import | 새 파일 작성 시 추가하는 각 `import {Module}` 문에 대해 그 모듈의 알려진 심볼이 파일 내에서 grep 0건 | 형제 파일 패턴 답습 의혹 (cargo-cult). 이유: 형제 파일의 import는 형제 파일의 *사용 심볼*이 정당화한 결과이며, 새 파일은 *자신의 사용 심볼*로 자체 정당화 필요 |

### 2. Lessons-to-Module Pipeline 도구화 (`/fz-manage reflect-to-module`)

메모리 17차(Reflection Gap)에 부분 응답 — 누적 교훈을 *수동 반영*하던 lag를 도구화로 해소.

**5-Step 절차**:
1. **Memory Parser** (`scripts/parse_memory.py`, 280줄): frontmatter + 본문에서 12 필드 추출 (id, applied_location, meta_family, signals 등)
2. **Relevance Scorer v2.1** (`scripts/score_relevance.py`, 270줄): 후보 모듈에 5 component 점수 계산 (trigger_match 0.55 / cascade_match 0.65 / lint_tool 0.70 / semantic 0~0.25 / symmetry_bonus 0.20). threshold 0.70 이상 selected.
3. **Suggestion Generator** (`prompts/generate_suggestion.md`): Claude prompt template — 5 type 분류 (catalog_addition / section_add+procedural_call / bidirectional_extension / perspective_addition / file_creation) + diff 제안
4. **Reviewer** (Codex micro-eval): 각 제안 외부 검증 (verdict: agree/disagree/partial/needs_verification) — O3 (Generator ≠ Evaluator) MUST 충족
5. **사용자 승인 + 적용**: 자동 적용 금지 (메모리 23차 self-review blind spot 방어)

`fz-manage/SKILL.md`에 `reflect-to-module` 서브커맨드 통합 (allowed-tools 확장 + intent-triggers 추가).
