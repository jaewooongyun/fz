## 🆕 Feature Set B: Tier 2 — T2-B Sprint Contract Pattern (33d1363)

### 1. `modules/sprint-contract.md` 신규

Cross-skill + 5+ Step plan 시 발동되는 새 패턴:

```yaml
sprint_id: <task-id>
contract_owner: codex_architect
context: { intent, scope, constraints }
success_criteria:
  - id: SC-N
    measurable: true
    description: <binary 판정 가능>
    verification: <build / grep / test>
anti_criteria:
  - id: AC-N
    description: <발생 금지 패턴>
    detection: <탐지 방법>
review_pass_threshold:
  reflection_rate_min: 80
scope_boundary: { in_scope, out_of_scope, deferred }
```

**핵심 메커니즘**: Codex가 Lead plan 작성 **전에** SC를 commit → Lead가 자기 plan에 우호적으로 SC를 retrofit 불가. Self-preference bias 구조적 우회.

### 2. fz-plan Phase 0.7 신규

발동 조건:
- TEAM 모드 + 코드/계획 생산 작업: 권장
- 5+ Step Plan + Cross-skill 변경: 필수
- 단순 수정 (1-2 Step): 스킵
- 탐색만 / 문서만 변경: 스킵

### 3. 첫 dogfooding 실증 (v4.3.0 자체 codex-utilization plan)

본 릴리즈의 Codex Utilization Plan v1 작성 시점에 **T2-B 첫 발동**:
1. Codex 작성 SC (sprint-contract-codex.md) — 5 SC + 3 AC + scope_boundary
2. Lead 1차 agree (수정 없음)
3. Lead plan v1 작성
4. Codex verify → APPROVED_WITH_NOTES (SC 5/5 PASS, AC 3/3 CLEAN)
5. Implementation 진입

**ROI**: Plan iteration cycle 1 round 압축 (vs 일반 cross-model 평균 2-3 cycles). Codex 메타 통찰 인용: "가장 큰 위험은 Codex 활용 강화가 다시 verify-first 루프로 번역되어 33차 action default를 약화하는 것" — 자기 잠재 함정을 사전 식별.
