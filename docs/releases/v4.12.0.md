**Title**: TEAM → 네이티브 Workflow 전환 (Wave 0-3)
**Type**: MINOR
**Trigger**: "팀즈를 다이나믹 워크플로우로" 세션 — TEAM(TeamCreate+SendMessage) 모드의 상호 통신 오류 + 팀 정리 오류 불만 → pilot(fz-discover) → 확산(Wave 1-3) → 전 스킬 5종 전환 완결.

## TL;DR

TEAM 멀티에이전트 실행을 **네이티브 Workflow 결정적 스크립트**(`workflows/*.js` 5개, 1094줄)로 전면 전환. 5개 스킬(discover/search/review/plan/code·fix)의 멀티에이전트 경로가 스크립트 오케스트레이션으로 바뀌며, TEAM의 2대 오류 클래스(통신 유실·정리 실패)가 **구조적으로 불가능**해짐 — 데이터는 스크립트 경유, 팀 lifecycle 자체가 없음. 10 커밋, 17파일 +1336/−273. breaking change 0.

## Highlights

### 1. workflows/ 신설 — 5 스크립트 (feat)

| 스크립트 | 대체 대상 | 구조 |
|---------|----------|------|
| `discover-adversarial.js` (pilot) | Adversarial Constraint Discovery | lean 5-call / --deep 렌즈 3 fan-out → merge → chunk ≤4 평가 |
| `search-cross-verify.js` | Cross-Verify | 심볼/패턴 독립 병렬 → 교차 FP 제거 → 병합 + 등급(binary rule) |
| `review-live.js` | Live Review | arch+quality 병렬 → id-기반 교차 severity 조정 → counter DA → 스크립트 병합 |
| `plan-collaborative.js` | Collaborative Design | direction(조건부 반박 왕복) → 초안 → 3렌즈 병렬 → CC 교차 → 통합 → 재검증 (9-11 call) |
| `code-pair.js` | Pair Programming | impl changeset → 조건부 arch 검토 → Lead 적용+빌드 (Step당 1 invoke, 1-3 call) |

공통 계약: 스키마 강제 JSON(자동 재시도) / null 명시 분기(silent fail 불가) / `mode:'fallback'` → SOLO 폴백 / metrics(agentCalls·nullCount·fallbackCount·stagesCompleted) 반환 → §5.7 기록.

### 2. changeset 책임 재배분 — code-pair (feat)

에이전트는 디스크를 수정하지 않는다 — **changeset JSON**(exact syntax `newBody` + `oldAnchor`)만 반환하고, Lead가 anchor 검증 → 적용 → 빌드. 검증 안 된 Edit이 디스크에 닿지 않는 구조. 실전 2회: oldAnchor 5/5 정확(에이전트 자가 grep 유일성 검증 + Lead 재검증 양방향 작동).

### 3. 표준 패턴 3종 — skill-authoring §12 신설 (docs)

probe 실측 기반 규약:
- **OVERRIDE 블록**: P2P 통신/Lead 보고 지시 무효화 + 에이전트 정의의 Phase·ASD·메모리 컨텍스트 로딩 무효화 + 무관 폴더 금지 (invoke 오염 2건에서 도출)
- **args 방어 파싱**: scriptPath 호출 시 args가 JSON **문자열**로 도착(0-agent probe 실측) → `typeof` 분기 + fail-fast
- **agentType `fz:` namespace 필수** (S0 probe: `'plan-structure'` 실패 / `'fz:plan-structure'` 성공)
- 명명 워크플로우 자동 등록: `meta.name` → 스킬 목록 등장 실측 — scriptPath 없이 이름 호출 가능
