# Release v4.13.0 (2026-06-11)

**Title**: Template Authority Bias 방어 + 구조 결정 옵션 사용자 배선
**Type**: MINOR
**Trigger**: ASD-1802(TVING iOS 찜 PR2)에서 fz 풀 파이프라인(discover→plan→plan-review→code→review)이 외부 인간 리뷰어(siparoma) 지적 3건을 **0건 선행 포착** → --deep 실패 분석(에이전트 3+counter 1, RC 5개) → counter가 원안 8건을 4건으로 절단 → 사용자 갭 분석 요구("옵션을 나에게 제시하는가")로 G1·G3 추가 확정.

## TL;DR

참고 템플릿(사용자 본인 코드 포함)을 미러링할 때 구조 결정이 "이미 결정됨"으로 통과되던 편향(Template Authority Bias)을 4층으로 방어: **발화층**(F2 token·F3 트리거·F4 few-shot이 잡고) → **판정층**(F1 3축 Quick-Check가 대안 ≥2 + trade-off 강제) → **전달층**(G1이 PROCEED여도 옵션 테이블을 사용자에게 제시) → **분류층**(G3가 "미러링=단순 수정" 스킵 함정 차단). 적용 중 발견한 내장 self-test 상시 실패 버그도 수정. 6 커밋, 11파일. breaking change 0.

### 2. G1·G3 — 사용자에게 보이게 (feat)

실측으로 확인된 배선 결손: DirectionSchema가 대안 2개를 강제해도 PROCEED 경로에선 Stage 4에 미전달 + PlanSchema에 옵션 필드 부재 + 반환 계약에 `directionVerdict`만 — **옵션이 생산돼도 사용자 도달 보장이 없었음**.

- **8a** `workflows/plan-collaborative.js`: workflow 반환에 `directionAlternatives` 패스스루 (PlanSchema·에이전트 프롬프트 무변경 — 행동 불변 배선) + 헤더 계약 주석 동기화
