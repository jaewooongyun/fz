# Release v4.12.1 (2026-06-08)

**Title**: Serena MCP 플러그인 번들 + 미노출 도구 참조 정리
**Type**: PATCH
**Trigger**: "fz에 serena가 잘 동작하나" 점검 → serena 미연결 발견(uvx 1회 실행 ≠ 영구 설치) → 재설치(serena-agent 1.5.4) → "플러그인에 포함" 결정(Option A: `.mcp.json` 번들) + dead ref 정리.

## TL;DR

Serena MCP를 `.mcp.json`으로 플러그인에 번들하여 `claude plugin install fz` 시 자동 등록되게 함(공식 serena 플러그인 패턴 + `--context claude-code` 개선). 동시에 serena 1.5.4 `claude-code` 컨텍스트가 노출하지 않는 3개 도구(`search_for_pattern`/`find_file`/`list_dir`) 참조를 전 스킬·에이전트에서 `Grep`/`Glob`로 정리(24파일 60 edit, 전 범위 0 잔여). 4 커밋. breaking change 0.

## Highlights

### 1. Serena MCP 번들 — .mcp.json (feat)

repo root(`source: "./"` → 플러그인 root)에 신설:

```json
{ "serena": { "command": "uvx",
  "args": ["--from","git+https://github.com/oraios/serena","serena","start-mcp-server","--context","claude-code","--project-from-cwd"] } }
```

- `claude plugin install fz` 시 serena 자동 등록 — 수동 `claude mcp add` 불필요.
- 공식 serena 플러그인 포맷 기반 + 개선: `--context claude-code`(공식판은 desktop-app default라 Claude Code에 비최적) + `--project-from-cwd`.
- 런타임 `uv` 필수(플러그인이 보장 못 함) — README prerequisite에 명시. serena 부재 시 fz는 Grep/Glob 4-tier fallback으로 graceful degrade(하드 실패 없음).

### 2. 미노출 도구 참조 정리 (refactor)

serena 1.5.4 `claude-code` 컨텍스트는 22개 도구를 노출 — 심볼·메모리 도구는 전부 포함하나 `search_for_pattern`/`find_file`/`list_dir`(파일·텍스트 검색류)은 제외(클라이언트 native Grep/Glob 가정). fz가 옛 도구셋(0.1.4) 기준으로 선언했던 참조를 전면 정리:

- `search_for_pattern`→`Grep`, `find_file`/`list_dir`→`Glob`/`Read`
- skill allowed-tools 10 + agent tools 7(+fz-memory 공유줄) + 본문/prose 지시 + fz-search 예시블록·에러표 + skill-troubleshooting 폴백체인 재번호
- 24파일 60 edit. frontmatter 무결성 검증 + 전 범위 grep 0 잔여
- 심볼 도구(`find_symbol`/`find_referencing_symbols`/`replace_symbol_body`/`rename_symbol` 등)·메모리 도구는 노출되므로 fz 핵심 기능 무손상

### 3. README prerequisite 교정 (docs)

## 활성화 (사용자)

GitHub 발행 후 설치본 반영엔 2단계 필요 — 설치본은 `github.com/jaewooongyun/fz`의 별도 클론이라 push만으론 안 바뀜:

1. `claude plugin update fz` (version bump 4.12.0→4.12.1 덕에 update 인식 — 누락 시 "already at latest" 스킵)
2. `/reload-plugins`(또는 재시작) → serena MCP 서버 최초 승인

## Breaking changes

없음. serena 부재 환경은 기존 Grep/Glob fallback 유지, 모든 스킬 인터페이스 동일.

## 알려진 제약

- **`uv` 런타임 의존**: 플러그인이 설치 보장 못 함 — `brew install uv` 안내. 미설치 시 serena 미가동(단 fallback으로 하드 실패 없음).
- **uvx cold-start**: 첫 실행 시 git build 지연(이후 캐시).
- dead ref enumeration이 `search_for_pattern({...})` 형태를 2회 누락 후 grep 전수로 0 확정 — 메모리 32차(probe coverage gap) 재발 기록.
